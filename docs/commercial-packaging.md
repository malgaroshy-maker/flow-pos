# دليل التوزيع التجاري وإصدار التراخيص — منظومة Flow (V1.4.0)

هذا المستند هو **الدليل الرسمي الموجه لمالك المنظومة والفريق الفني** لكيفية تجميع حزمة التوزيع التجاري، إصدار التراخيص غير المتصلة بالإنترنت للعملاء، وتثبيت المنظومة في أجهزة المحلات.

---

## 1. فلسفة التوزيع والأمان

- **منتج مغلق مرخّص:** تُباع المنظومة كمنتج بررمجي مغلق (Closed Software) مرخص لكل محل بناءً على كود التفعيل الخاص بالحاسوب الشخصي.
- **بدون إنترنت 100%:** جميع عمليات التحقق من التراخيص وتدقيق التوقيع التشفيري تتم محلياً دون أي اتصال بخوادم خارجية.
- **تشفير التوقيع (Ed25519):** يستخدم النظام توقيعاً رقمياً غير متماثل (Ed25519) لمنع أي تلاعب بمفتاح الترخيص أو تغيير اسم العميل أو تاريخ الصلاحية.
- **حماية الكود المصدري:** يتم تجميع وتصغير خادم الـ Node.js باستخدام مكتبة `esbuild` في ملف واحد (`server/dist/server.js`) مع إخفاء الكود المصدري وإبعاد أداة توليد التراخيص (`scripts/keygen.ts`) والمفتاح الخاص خارج الحزمة.

---

## 2. كيفية بناء حزمة الإنتاج التجاري

تشغيل أمر البناء الشامل من جذر المشروع:

```bash
npm run build
```

يقوم هذا الأمر بتجميع:
1. **الخادم الخلفي:** فحص التجميع وتوليد `server/dist/server.js` المصغّر.
2. **الواجهة الأمامية:** تجميع وتصغير تطبيق الويب في `web/dist`.

---

## 3. كيفية إصدار ترخيص لعميل جديد (Vendor Keygen)

أداة توليد التراخيص تقع في `scripts/keygen.ts` وتستلزم استخدام المفتاح الخاص بالمالك (`vendor_private.pem`).

### أ. الحصول على كود الجهاز من العميل (Machine Code)
عند تشغيل المنظومة لأول مرة لدى العميل دون ترخيص، تظهر شاشة التفعيل وتعرض كود الجهاز المكون من 16 حرفاً، مثال:
`A1B2-C3D4-E5F6-7890`

### ب. إصدار مفتاح ترخيص تجاري دائم
من جهاز المالك (خارج حزمة العميل):

```bash
npx tsx scripts/keygen.ts --machine A1B2-C3D4-E5F6-7890 --customer "مقهى ومطعم الشاطئ" --type commercial
```

### ج. إصدار ترخيص تجريبي محدد بـ 30 يوماً
```bash
npx tsx scripts/keygen.ts --machine A1B2-C3D4-E5F6-7890 --customer "تجربة - محل السلام" --days 30 --type trial
```

يظهر في الشاشة مفتاح الترخيص المشفر المشترك (Base64)؛ يتم إرساله للعميل عبر الواتساب أو الفلاشة ليلصقه في شاشة التفعيل.

---

## 4. مسارات البيانات والملفات لدى العميل

- **ملفات النظام البرمجية:** تُثبت في `C:\Program Files\FlowPOS`
- **قاعدة البيانات والتراخيص والصور:** تُحفظ في `C:\ProgramData\FlowPOS`
  - `pos.db` (قاعدة بيانات SQLite)
  - `license.lic` (ملف الترخيص المفعل)
  - `uploads/` (صور المنتجات)
  - `backups/` (النسخ الاحتياطية)

عند تثبيت تحديث جديد (إصدار أحدث)، يتم استبدال الملفات البرمجية فقط مع المحافظة التامة على مجلد البيانات `ProgramData`.

---

## 5. خطوات الفحص والتحقق في بيئة وهمية (VM Test Checklist)

1. **النسخ النظيف:** تثبيت حزمة التوزيع على جهاز ويندوز جديد أو بيئة وهمية (Clean Windows VM).
2. **التحقق من حظر الوصول:** فتح المتصفح على `http://localhost:3001` والتأكد من ظهور شاشة تفعيل الترخيص وحظر باقي المسارات.
3. **التفعيل:** إدخال مفتاح ترخيص صادر بكود الجهاز والتحقق من فتح جميع شاشات المنظومة وتدفق العمل.
4. **الاتصال عبر الشبكة المحلية (LAN):** الاتصال من جوال أو كاشير آخر عبر الأيباد على IP الخادم المحلي والتأكد من فتح شاشة الكاشير بسلاسة.
